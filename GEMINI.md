# Connect Africa Development Protocols

## Production-First Mandate
- **No Stubs/Placeholders:** Every line of code must be production-ready. Stubs, TODOs, or placeholder logic are strictly prohibited in the codebase.
- **Full Implementation:** Every feature must be fully implemented, documented, and tested according to the `Docs/` specifications before moving to the next milestone.
- **No Shortcuts:** Shortcuts that bypass security, validation, audit logging, or architectural requirements are forbidden.
- **Roadmap Discipline:** Treat the `PRODUCTION_ROADMAP.md` as the source of truth. Checkmark milestones ONLY after completion, full verification, and commitment to git.
- **Investigation Before Modification:** Never remove files, imports, or code without thoroughly investigating their purpose and ensuring safe removal/refactoring.
- **Commit Protocol:** After the completion and verification of each milestone, perform a git commit.
- **Zero-Tolerance for MVP-Only mindset:** Build for scalability and production quality from day one.
